This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
# Install dependencies (first time / after Gemfile changes)
bundle install

# Serve locally with live reload
bundle exec jekyll serve

# Build the site
bundle exec jekyll build

# Serve with drafts visible
bundle exec jekyll serve --drafts
```

After `bundle exec jekyll serve`, the site is available at `http://localhost:4000`.

Deployment is via GitHub Actions (`.github/workflows/deploy.yml`) — pushing to `master` triggers a build and deploys to GitHub Pages automatically. The Pages source must be set to "GitHub Actions" in repo settings.

## Architecture

Jekyll 4 static site blog deployed to GitHub Pages via GitHub Actions.

- `_posts/` — Blog posts in Markdown with YAML front matter. Filenames must follow `YYYY-MM-DD-Title.markdown`. The `defaults:` block in `_config.yml` injects `layout`, `author`, and `comments: true` automatically, so only `title` is required in post front matter.
- `_layouts/` — `default` (HTML shell), `page` (static pages), `post` (blog posts with reading time/tags/subscribe/comments), `archive` (generated by jekyll-archives for tag/category pages).
- `_includes/` — `head.html`, `header.html`, `footer.html`, `reading-time.html`, `subscribe.html` (Buttondown), `comments.html` (Giscus), `dark-mode.html` (JS toggle), `tag-list.html`.
- `_sass/` — `_variables.scss` (CSS custom properties for light/dark themes), `_base.scss` (reset/typography), `_layout.scss` (structural layout), `_components.scss` (subscribe form, reading time), `_syntax-highlighting.scss` (code blocks + dark mode overrides).
- `_config.yml` — Plugins: `jekyll-feed`, `jekyll-seo-tag`, `jekyll-sitemap`, `jekyll-archives`. Giscus and Buttondown config live here.
- `tags.md` — Root-level tag cloud page at `/tags/`. Tag archive pages at `/tags/:name/` are auto-generated by `jekyll-archives`.

## Dark Mode

CSS custom properties on `:root` (light) and `[data-theme="dark"]` (dark) drive theming. JS in `_includes/dark-mode.html` reads/writes `localStorage` and sets the attribute on `<html>`. An inline script in `head.html` applies the saved theme synchronously before paint (prevents flash).

## Adding Tags to Posts

Add `tags: [ruby, rails, aws]` to post front matter. The tag cloud at `/tags/` and per-tag archive pages at `/tags/:name/` update automatically.

## Third-party Integrations

- **Buttondown** — subscribe form in `_includes/subscribe.html`. Username: `thelucasmoore`.
- **Giscus** — comments in `_includes/comments.html`. Requires `repo_id` and `category_id` values from giscus.app in `_config.yml`. GitHub Discussions must be enabled on the repo.

## Notes

- Kramdown is the markdown parser. In kramdown, `#` headings inside posts render as `<h1>` — use `##` or `###` for section headings within post body.
- `_config.yml` changes require restarting `jekyll serve` to take effect.
- The Sass `@import` deprecation warnings during build are harmless — Jekyll 4's bundled Sass still supports it.
